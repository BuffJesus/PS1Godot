## TL;DR

The cube round-trip works now: bake in Blender via Cycles, click
**Export to Godot**, click **Run on PSX**, the bake ships through
to PSX hardware. This had been silently broken across three
separate failure points; all three are fixed.

Slot D1 (automatic static-mesh batching) is disabled via a
kill-switch — three correctness fixes landed but the merged batches
still don't render correctly on PSX. The full diagnosis + rework
scope is in `ROADMAP.md`. Until the surface-index remap lands, every
static mesh exports as its own GameObject (pre-Slot-D1 behaviour, no
visible regression).

### Slot D1 static-mesh batching disabled (1 commit)

- **`9934bd1`** slot d1: disable static batching pending rework —
  `s_enabled = false` kill-switch at the top of
  `StaticBatchOptimizer.cs`. Three correctness fixes are kept in
  place because they're prerequisites for the eventual rework:

  1. Parentless synthetic batch node — `Node3D` outside the scene
     tree, so `GlobalPosition`/`GlobalBasis`/`GlobalTransform` returns
     `Transform3D()` (identity) and the GO writes at world (0,0,0).
     Fixed in `SplashpackWriter.cs` + `PS1GodotPlugin.cs` with
     `IsInsideTree() ? GlobalTransform : Transform` fallback.

  2. Bucket key has no spatial component — all world meshes sharing a
     texture page collapsed into one batch regardless of distance,
     overflowing the int16 fp12 vert envelope (~±32 Godot units around
     the anchor). Fixed by adding a 24 m world-AABB-centre cell to the
     bucket key + a per-merge safety guard that bails out if combined
     extent still exceeds the envelope.

  3. Y/Z flip in `TransformVertex` — PSX vertex storage flips Y/Z
     (left-handed); the optimizer's decode + encode both skipped the
     flip. Identity-rotation members coincidentally worked because the
     missing flips cancel; rotated members produced scrambled verts.
     Now both ends apply the flip.

  Remaining suspect (not yet patched): `combinedTexIndices.AddRange`
  appends each member's `SurfaceTextureIndices` without offsetting,
  so triangles from member B sample member A's surface slot at
  runtime. Needs surface-remap pass + texture-correctness check in
  `MeshFormatRoundTripVerifier` before re-enabling.

  Memory pin saved at
  `memory/project_static_batch_disabled.md` so the next session
  doesn't re-walk the diagnosis.

### Blender ↔ Godot baked-lighting round-trip (5 commits)

The big arc this session: testing the Blender bake → Cycles →
Export to Godot → Run on PSX flow with the default cube. Three
silent-failure modes turned up; all three fixed:

1. **`9ce6e3e`** blender(export): preserve baked vertex colors
   through gltf — the gltf addon was silently dropping the bake
   because the default Blender material doesn't reference vertex
   colours in its node tree. Set
   `export_active_vertex_color_when_no_material=True` so the bake
   rides through to Godot regardless.

2. **`031e283`** blender(panel): always surface "Create PS1 Material"
   — the PS1 Material panel's `poll()` hid the entire panel (and
   the Create button) when the active object had zero material slots.
   Authors who deleted the cube's default material before baking lost
   the only visible path forward. Panel now shows whenever a mesh is
   selected; collapses to a "No material assigned" hint + Create
   button when no material exists.

3. **`bbe97ed`** blender(bake): default to Diffuse + warn on saturation
   + auto-show colors — three UX fixes for the Cycles bake → "every
   face white" trap. (a) Default Cycles Bake Mode is now Diffuse
   instead of Combined. Combined sums direct + indirect + emission
   and pegs at the PSX 0.8 ceiling on every face when the scene has
   any decent light. (b) Bake operator now WARNs (in addition to
   INFO count) when 50%+ of vertex loops saturated. (c) Both Cycles
   and scene-lights operators flip every 3D viewport's color_type to
   VERTEX after a successful bake so authors can confirm the result.

4. **`e7a0825`** blender(bake): force viewport to Solid + Vertex
   unconditionally — earlier viewport-switch guarded on
   `shading.type == "SOLID"` which silently no-op'd when Blender was
   in Material Preview / Rendered mode. Now sets both type and
   color_type unconditionally; reports `viewport set to Solid +
   Vertex Color (N viewport(s))` on success or warns when no 3D
   viewport found.

5. **`c4c2b8d`** exporter(mesh): read intrinsic COLOR array as bake
   fallback — **the actual blocker.** `PSXMesh.FromGodotMesh` +
   `AppendFromGodotSurface` only consumed the per-instance
   `PS1MeshInstance.BakedColors` override and ignored the mesh's own
   `ArrayType.Color` channel. Result: the entire Blender bake →
   glTF → Godot → PSX path was silently broken — the bake landed in
   the imported mesh's COLOR array, the exporter threw it away, the
   PSX scene rendered with FlatColor white. Fix: per surface, prefer
   `BakedColors` override (Godot-side bake), fall back to
   `mesh.Color` array (Blender-side bake), final fallback to flat
   colour.
