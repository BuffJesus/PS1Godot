## Where the tooltip plugin currently is

Architecture:

```
addons/ps1godot/ui/PS1InspectorTooltips.cs   ŌåÉ new plugin
addons/ps1godot/PS1GodotPlugin.cs            ŌåÉ registers it via AddInspectorPlugin
PS1Godot.csproj                              ŌåÉ <GenerateDocumentationFile>true</> (task #1)
```

Flow:

1. `<GenerateDocumentationFile>true</>` makes the C# compiler emit
   `PS1Godot.xml` next to the DLL. Verified: 262 KB sidecar at
   `.godot/mono/temp/bin/Debug/PS1Godot.xml` containing every `///`
   summary.
2. On plugin construction, `PS1InspectorTooltips` parses the XML into
   a `(className, propName) ŌåÆ summary` dict. The path resolution tries
   `Assembly.Location` first, then falls back to
   `res://.godot/mono/temp/bin/{Debug,ExportDebug,Release}/PS1Godot.xml`.
3. `_CanHandle(obj)` returns true when `obj.GetType().Name` is in the
   dict.
4. `_ParseBegin(obj)` defers a sweep that walks
   `EditorInterface.Singleton.GetInspector()` recursively, finds every
   `EditorProperty`, looks up `GetEditedProperty()` in the dict, sets
   `TooltipText`.

The very latest fix (commit b590a3e) swapped `XmlReader` for
`XDocument` because `ReadElementContentAsString` threw on summaries
with mixed content. As of this fix the path is theoretically end-to-end
working ŌĆö **needs in-editor verification**.

## Wrinkles & gotchas to remember

### C# DLL doesn't hot-reload
Memory: `project_godot_dll_hot_reload.md`. Every change to the plugin
needs a Godot restart. Tomorrow you'll restart at least once for the
tooltip XML fix.

### CRLF line endings
The `.cs` files are CRLF. PowerShell scripts I used preserved that;
Python's default would have flipped to LF and produced massive
spurious diffs. If you script anything similar, use the PowerShell
patterns I used (`[System.IO.File]::ReadAllText` + UTF8Encoding(false)
+ explicit CRLF in `-join`).

### `[ExportGroup]` placement vs `///` comments
Roslyn detaches `///` from the next member if `[ExportGroup(...)]`
appears between the comment and the property. The fix is to put the
doc-comment ABOVE the `[ExportGroup]`, then `[Export]` after. The
sweep in commit `001c3c8` corrected this for every site that the
build flagged with CS1587. New properties should follow the same
order: `///`, `[ExportGroup]`, `[Export]`.

### Inspector configuration-warning Call needs HasMethod gate
Godot's `Call("_get_configuration_warnings")` raises a hard engine
error on every node that *doesn't* override it. The C# `try/catch`
doesn't catch it (it goes through `GD.PushError`, not a managed
exception). The dock's traversal in `PS1GodotDock.cs:438` gates with
`HasMethod` to silence the noise. If you add new Call-by-snake-case
patterns elsewhere, do the same.

### Auto-camera-export Lua preamble
Every Camera3D in the active scene auto-emits a `Camera.LoadFromExport`
table prepended to the scene's Lua source. The preamble is ASCII-only
(psxlua's tokenizer doesn't reliably handle UTF-8 multibyte chars),
and uses `CultureInfo.InvariantCulture` for number formatting. If you
add fields, keep both rules.

### .tscn `Transform3D` row-major basis (memory)
Memory: `project_godot_tscn_basis_row_major.md`. The 9 basis floats in
a `.tscn` Transform3D are row-major. Reading them as columns gives
flipped y_basis/z_basis and ~180┬░ wrong Euler signs. Cost us a bad
camera auto-export 2026-04-30 ŌĆö already fixed; don't re-introduce.
