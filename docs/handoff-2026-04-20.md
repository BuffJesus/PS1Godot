## Open bugs reported but not yet fixed (priority order)

### 1. `Entity.SetPosition` signature was wrong (FIXED but not pushed)

`test_logger.lua` was calling `Entity.SetPosition(playerMesh, p.x, p.y, p.z)`
— the runtime takes a Vec3 table as the second arg, not 3 numbers.
Silent no-op via `if (!lua.isTable(2)) return 0;` left the player
mesh stuck at (0, 0, 0) regardless of runtime player position.

Fixed in `test_logger.lua` to `Entity.SetPosition(playerMesh, p)`. Stage
+ commit + push first thing.

### 2. Player mesh "way too large" on PSX

Humanoid renders at sane size in Godot's editor viewport but is huge
on PSX. Hypothesis: the PSXMesh exporter reads raw vertex positions
from `mesh.GetSurfaceArrays()[Vertex]` and divides by `gteScaling=4`,
but **doesn't apply the MeshInstance3D's transform scale**. If the FBX
imported with a non-1.0 internal scale (e.g. cm units → factor 100)
that's compensated by an instance transform scale on the Godot side,
the exporter sees the raw (huge) vertex values.

Fix candidate: in `PSXMesh.FromGodotMesh()`, multiply each vertex by
`node.Scale` (or `node.GlobalTransform.Basis.Scale`) before the
gteScaling divide. Plus emit a warning when `Scale != Vector3.One`
since combining that with affine warp can look weird.

File: `godot-ps1/addons/ps1godot/exporter/PSXMesh.cs:17` —
`FromGodotMesh()`.

If audio still silent after (1) is fixed: check whether per-clip

## Important context to remember

- **psxlua is integer-only.** No float literals (`0.00001`, `2.4`).
  Use `Convert.IntToFp(rawValue)` if you need fractional fp12 values.
- **tscn does NOT support `#` comments.** Adding `#` lines corrupts
  the parser and can drop nodes silently. Notes go in commit messages
  or adjacent .md files.
- **Entity.SetPosition takes a Vec3 table.** Same for any other
  function that takes a Vec3 — don't pass 3 numbers.
- **Default material `ps1_default.tres`** must stay texture-free. It
  gets auto-assigned to every new PS1MeshInstance via
  `_EnterTree()`. Floor uses the dedicated `ps1_floor_grass.tres`.
- **Cutscene.Play vs PS1AudioEvent timing.** The runtime's
  `m_frame` and Lua's onUpdate-tick advance at different rates
  (60 fps onUpdate vs 30 fps cutscene-frame). For text+audio sync
  during cutscenes, fire BOTH from Lua at the same `cutsceneFrame`
  counter (which only increments while `Cutscene.IsPlaying()`).
  Don't use PS1AudioEvent + separate Lua text reveals.
- **Demo layout right now**:
  - PS1Player spawn: `(0, 2, 8)` facing -Z
  - PS1TriggerBox: `(0, 1, 4)` HE=(3, 2, 1) — gate
  - Cube (green, spinning): `(-4, 1, -2)` interactable Triangle
  - Cube2 (checkered, bouncing): `(4, 1, -3)` interactable Triangle
  - CheckeredZone trigger: wraps Cube2
  - Player (humanoid, T-pose): tracks runtime player position via Lua
