## TL;DR

The Blender ↔ Godot pipeline is now genuinely **first-class authoring
in both tools, with full geometry + metadata round-trip in both
directions**. The original architecture memo's load-bearing claim
("Blender add-on is a peer authoring path") is now true rather than
aspirational.

Concretely:

- **Cross-tool round-trip (both directions, both metadata + geometry).**
  Click `Export to Godot` in Blender → `.glb` + JSON sidecars land in
  the Godot project; Godot's import scanner picks them up. Click
  `Edit Mesh in Blender` in Godot → mesh extracts as `.glb`,
  `pmi.Mesh` auto-rebinds to the imported asset, Blender opens.
- **Lighting parity in both tools.** 6 Blender bake operators (Lambert,
  scene-lights with shadow casting + color temperature, ambient tint,
  Cycles `COMBINED`/`DIFFUSE`/`AO`, clear) + 2 Godot bake operators
  (scene-lights Lambert, vertex AO via raycast). All clamp to PSX 0.8
  semi-trans ceiling.
- **Static batching at export.** Slot D1 minimum tier — auto-collapses
  meshes sharing `(DrawPhase, ShadingMode, AlphaMode, AtlasGroup,
  Translucent, texture_page_id)` into single GameObjects. Blender-side
  validator surfaces "would batch N → M" hints at author time.
- **Validation pipeline** with 5 reporters (texture / audio /
  animation / UV / mesh-dedup) feeding a single severity-tiered
  dock summary (✓ / ▲ / ✗) with click-to-focus per-mesh navigation.

## Key memory-pinned facts to carry forward

- **Wire-identifier stability is the round-trip contract.** Enum
  member names in `tools/blender-addon/.../properties.py` and
  `godot-ps1/addons/ps1godot/exporter/PS1Metadata.cs` must match
  character-for-character. Adding values is safe; renaming is not.
  See `memory/project_blender_addon_phase1.md`.
- **The 0.8 PSX 2× semi-trans ceiling is load-bearing.** Bake to
  1.0 and meshes tagged `Translucent` later white-out. Match
  SplashEdit `Runtime/PSXLightingBaker.cs:79-83`.
- **Renderer rooms-path lambda must carry pool pointers across
  cells** by reference. `lastVerts/lastFaces` need by-ref like
  `lastObj`; locals null-deref on first triref of a new cell
  sharing the previous object. See `commit 3639533` +
  `memory/project_renderer_lambda_state_carry.md`.
