# PS1Godot session handoff — 2026-04-28

Seven commits, all on `main`, all pushed. Phase 3 cleared the bulk of
its remaining items in one focused session — the splashpack triplet
now ships with a loading screen, F5 in Godot lands the game in
PCSX-Redux without leaving the editor, the inspector previews
PSX-quantized textures live, the validator stack catches
near-duplicate CLUTs + over-stacked translucent UI rects before they
hit hardware, and a new bottom-panel VRAM viewer (with hover
tooltips) shows authors exactly what's eating their 1024×512 budget.

## TL;DR

The end-to-end "open Godot, press F5, play on PSX" loop is now real:

1. Author picks scene, presses F5 in Godot
2. Plugin's `_Build()` override fires `OnRunOnPsx` (export → ISO build
   if XA-routed → launch PCSX-Redux)
3. Loading screen renders (black bg + green progress bar) while VRAM /
   SPU / splashpack files load, progress bar driven by the runtime's
   per-step checkpoints
4. Game scene boots, no leftover overlay

Plus a quieter authoring win: every PS1Sky and PS1UIElement(Image)
now shows a "PSX Preview (quantized)" panel in the inspector,
running the same quantizer the exporter ships with — what authors
see is what lands on hardware.

### `7687437` — Loading screens end-to-end (LoaderPack writer)

`scene_N.loading` now ships alongside the splashpack triplet whenever
a UICanvas is marked `Residency=LoadingScreen`. Carries its own
atlas/CLUT/font subset matching the main scene's VRAM positions so
the canvas's UV coordinates stay valid across the loading-screen →
splashpack handoff.

**Companion changes**:
- `PS1UIElementType.Progress` (type 3) with `BgColor` + `InitialValue`
  fields wired through to the runtime's `UIProgressData` typedata
- `SceneCollector` warns when a LoadingScreen canvas has no Progress
  element named "loading"
- `WriteUITableBlock` extracted from `WriteUISection` so both writers
  share the canvas/element/string serializer
- ISO bundler picks up the optional `.loading` sidecar as `SCENE_n.LDG`

**Bug found and fixed mid-session**: splashpack writer was emitting
LoadingScreen-residency canvases into the gameplay UI section *too*,
not just the LoaderPack. Result: canvas double-rendered after scene
load completed, sticking on screen forever. Fixed via
`GameplayCanvases()` filter at four sites — header count, write gate,
table block, UI 3D model section. Empirically verified by the user
on PSX after the fix.

### `37de85f` — F5 routes to Run-on-PSX

`PS1GodotPlugin._Build()` override hooks Godot's Play button into the
existing `OnRunOnPsx` pipeline. Returning false is the only way to
suppress Godot's own scene runner — the game runs in PCSX-Redux, not
in a Godot window — but that pops a "Project run failed" toast.
Editor log proactively explains the toast is harmless before it
appears.

ROADMAP also caught up on `[ ]` ISO build path (was actually shipped
2026-04-27 with `build_iso.py`); disc-layout-aware seek optimization
gated on Phase 2.5 chunk streaming becomes a child bullet.

### `3f89a51` — Quantized texture preview in inspector

`EditorInspectorPlugin` (in `addons/ps1godot/ui/`) attaches a "PSX
Preview (quantized)" panel to PS1Sky and PS1UIElement(Image). Runs
the same quantizer pipeline the exporter ships
(`ImageProcessing.Quantize` + 5-bit-per-channel clamp from
`VRAMPixel`). Auto-refreshes on Texture / BitDepth / Type edits via
the inspector's `property_edited` signal — no refresh button. Hides
the panel when a UIElement's Type switches away from Image so the
preview doesn't orphan under unrelated properties.

**Pending**: PS1MeshInstance / PS1MeshGroup previews. Those textures
flow through `material_override` / per-surface materials, not a
direct field; mesh preview means probing the material chain to find
the active albedo. Worth doing once the authoring story for mesh
materials is settled (currently authors edit `material_override`
manually).

### `5f93a94` — Near-duplicate CLUT + decal stack validators

Two small validators round out the Phase 3 scene-validation story so
the dock's "Last export" summary catches more authoring foot-guns
without requiring a PSX run.

**Near-duplicate CLUT** extends `TextureValidationReport`: pairwise
Euclidean distance over slot-aligned 5-bit RGB palettes, threshold
4.0/31 average per channel. Catches paletted-texture pairs whose
CLUTs could merge into one VRAM slot (~32 B for 4bpp, ~512 B for
8bpp saved per pair). Slot-by-slot comparison so the savings
estimate assumes no per-pixel index remap; reordered-but-similar
palettes still surface as a hint.

**Decal stack** (A3) — new `DecalValidationReport`. Walks each
visible-on-load UICanvas, counts how many other Translucent rects
each Translucent rect overlaps, warns when stack depth > 6 (the
threshold where serial blending starts to dominate PSX fillrate).
Names the worst element. Excludes LoadingScreen canvases (their
fillrate is moot during the load) and world-space meshes (3D AABB
overlap doesn't reliably predict screen overlap without camera info).

### `c18c900` — VRAM viewer dock — base layout view

`PS1VRAMViewerDock` (bottom panel "PS1 VRAM") + `PS1VRAMGrid`
custom-`_Draw` Control render the 1024×512 VRAM grid at every
export. Reserved regions (framebuffers + font column) in gray,
atlas footprints color-coded by bit depth (4bpp green / 8bpp blue
/ 16bpp orange), per-texture sub-rects on top, CLUT strips in
yellow. Header line shows scene name + percent of usable VRAM
consumed; legend chip explains the colors.

Custom `_Draw` rather than Image+TextureRect on purpose — opens the
door to per-tpage zoom / multi-scene picker / asset-residency
filters in v2 without reworking the rendering pass. `VramSnapshot`
decouples the dock from live `SceneData` / `VRAMPacker` (which churn
between exports), captured once `SplashpackWriter.Write` returns.

ROADMAP base entry checked off; the original amendment items
(per-scene residency view, texture-page grouping, OT-pressure
readout) become explicit child bullets — they need either asset
residency tagging (REF-GAP-1/-2/-3) or gameplay-frame metrics that
aren't available at export time.

### `9244de7` — VRAM viewer hover tooltips

`PS1VRAMGrid._GetTooltip(Vector2)` maps screen-space hover position
back to VRAM coords and hit-tests against the snapshot's rects
(textures → CLUTs → atlases, in that order so nested rects tooltip
correctly). Source-pixel width undoes the bit-depth pack ratio so
tooltips show authors the original art dimensions, not the
VRAM-cell count. CLUT strips get a 1px hover-padding band since
they're 1px tall on the source. Reserved regions tooltip too —
authors who hover over a framebuffer or the font column get a hint
instead of nothing.

## Where things stand at session end

### Working end-to-end

- **Loading screens**: title.tscn → `Scene.Load(3)` → progress bar
  fills 0→25→100% as VRAM/SPU/splashpack each load → monitor scene.
- **F5 in Godot** → editor exports → builds ISO if XA-routed → boots
  in PCSX-Redux. One keystroke from edit to play on hardware.
- **Inspector preview** for any PS1Sky / PS1UIElement(Image): tweak
  BitDepth, see the quantized result update live.
- **Validator stack** (texture + audio + animation + UV + decal +
  mesh-dedup) all run on every export and feed the dock summary
  line.

### Phase 3 remaining

Two main items left, plus a handful of child bullets. None are
blockers for the "author can ship a PS1 game" workflow — that's now
functional — they're each polish or developer-experience improvements
on top.

1. **Lua hot-swap over PCdrv** — Half-day, but cross-cutting (runtime
   + plugin). Plugin side: detect single `.lua` change, write it,
   signal runtime. Runtime side: receive "reload script X" via
   PCdrv, replace bytecode, reset script state without re-running
   `onCreate`. Needs PSX-side verification — high regression risk
   if the VM hot-reload corrupts state mid-game.

2. **Project template (`PS1 Game`)** — Installable into Godot's
   project manager. Polish-tier — defer per user preference until
   core is fully usable.

Phase 3 child bullets still open:
- **VRAM viewer per-scene residency view** (`REF-GAP-1`/`-2`/`-3`)
  — environment / character / UI / effects broken out. Needs
  residency tagging on assets first.
- **VRAM viewer texture-page grouping view** — highlight which
  textures share a tpage, show tpage-switch count per draw call.
- **VRAM viewer OT-pressure readout** — object count vs
  `PS1Scene.MaxActors`, texture-page switches per frame vs
  `MaxTexturePages`. Gameplay-frame metrics, needs runtime feedback.
- **VRAM viewer multi-scene picker** — dropdown to pick which
  scene's snapshot to display.
- **Texture-reuse "unique-atlas per mesh"** detection — needs richer
  `AtlasGroup` signal than current bucket.
- **PS1MeshInstance / PS1MeshGroup texture preview** — needs material
  chain probing (see `3f89a51` notes).
- **F5 to play debugger attach** — current implementation just
  launches; doesn't attach a C# debugger or tail printf into Godot
  Output dock. Listed as separate item under the main `[x]`.
