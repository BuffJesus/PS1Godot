## Resume point — UI designer drag doesn't move mesh on PSX

**Status:** diagnosed but not fixed. Hand this off as task #1 tomorrow.

### What's working
- UI Designer drag moves the rect in the 2D canvas preview ✓
- Anchor-aware delta flip — cursor tracks correctly on all 9 anchors ✓
- Inspector X/Y update and persist through export ✓ (confirmed live)
- Splashpack serializes the updated `screenX`/`screenY` ✓
- Runtime `renderUIModels` is called with the correct values ✓
- Per-model OFX/OFY math is correct ✓ (log showed `center=(59,120)
  ofxRaw=3866624 ofyRaw=7864320` for `screenXY=(11,72)` — arithmetic
  is exact: `59 = 11 + 96/2`, `3866624 = 59 << 16`)

### What's broken
The GTE `write<Register::OFX>` / `write<Register::OFY>` calls emit
`ctc2` via `asm volatile` and the logs confirm the computed raw values
are correct, **but the projected model still renders at screen-center
(160, 120).** The write isn't producing a visible effect.

### Leads for tomorrow
1. **Confirm the register actually changes.** Add a
   `readRaw<Register::OFX>` right after the write, `printf` the
   returned value. If it reads back as 10485760 (the default 160.0
   fp16), the write is being swallowed. If it reads back correctly,
   the issue is downstream of the register.
2. **Hardcode an extreme value.** Change my `write<Register::OFX>`
   to a constant far from 160 (e.g., `write(50 << 16)`). If the model
   doesn't shift at all, OFX is not the right lever for this
   projection path. If it shifts to an unexpected place, the math
   needs rethinking.
3. **Alternative: TRX translation.** Instead of shifting OFX, compute
   a view-space X/Y offset and add to TRX/TRY. The projection formula
   `SX = (H * MAC1) / SZ3 + OFX` means TRX contribution is
   depth-dependent, but for a fixed-distance orbit (`viewPos.z =
   currentDistFp12`), we can solve: `TRX = dx_pixels * dist / H`.
   That's a guaranteed-works fallback if OFX really doesn't wire
   through whatever GTE path processTriangle uses.
4. **Check rtpt kernel.** Grep `gte-kernels.hh` + `cop2 0x0280030` —
   rtpt *should* read OFX but psyqo's wrapper may have some hazard the
   examples sidestep implicitly. Compare to a torus/cube example that
   uses OFX at runtime.

### Pains that bit us authoring Lua today

Ordered by how much they hurt:

1. **Decimal literals crash the script at load.** `0.06` → `malformed
   number near '0.06'` → whole Lua chunk refuses. psxlua is NOPARSER +
   integer-only. Every first-time author hits this, with no
   breadcrumb to the fix.
2. **fp12 literacy tax.** Every *numeric* API (`Camera.Shake`,
   `Camera.ShakeRaw`, position math) needs the author to know
   `0.06 × 4096 = 246` or use the FixedPoint metatable `.new(int,
   frac)`. Today we shipped a parallel `Camera.ShakeRaw` just to
   sidestep the decimal trap.
3. **Undocumented PS1 conventions bite.** Y is inverted on PSX (+Y
   down); `Player.GetPosition()` returns the camera head in
   third-person (**not** the player body); 3rd-person rig is a fixed
   3-unit offset. Each of these cost us a round-trip today
   (bullet-from-camera, inverted melee hitbox).
4. **`Debug.Log` tanks framerate** when called on hot paths — PSX TTY
   is a synchronous UART write. R2 whiffs logging every press was
   perceptible stutter.
5. **No autocomplete / LSP** — ROADMAP design principle #4 says
   "generate EmmyLua stubs from `luaapi.hh` at plugin install". Still
   only a principle.

### Proposed attack order (low barrier of entry first)

Fix what *blocks first-time authors*. Polish after.

#### Task 1 — Decimal-literal pre-processor (biggest unlock)

**Goal:** authors write `Camera.Shake(0.06, 6)` and it works.

At export time (in `SceneCollector` / wherever Lua chunks get embedded
in the splashpack), run a lightweight pass over the Lua source that:
- Matches decimal literals via a simple state-machine lexer (not
  regex — Lua string literals may contain them).
- Rewrites each `0.06` → `FixedPoint.newFromRaw(246)` (or
  `FixedPoint.new(0, 246)` using the existing helpers in
  `third_party/nugget/psyqo-lua/src/lua.cpp:347`). Integer part +
  fractional raw fp12.
- Leaves integers, hex, scientific notation, and string contents
  alone.

Scope constraint: the exporter already has the Lua source bytes; it
doesn't need to interact with psxlua at all. This is a C# text
pass.

Files: add `godot-ps1/addons/ps1godot/exporter/LuaDecimalRewriter.cs`,
call from the Lua-embedding site in `SceneCollector`.

Test harness: add a couple of unit tests alongside
`MidiSerializerTests.cs` for the rewriter — it's pure text in / text
out, easy to check.

Document: one line in api-showcase / QUICKSTART saying "decimals work;
they're rewritten to fp12 at export".

#### Task 2 — Actionable error message on load failure

When `loadBuffer` in psxlua returns a parse error, currently the
runtime aborts with the raw Lua message. Add a string-match in
`luaapi.cpp` (or wherever we print the Lua error) that detects
`malformed number near '` and appends:
`(hint: psxlua is integer-only — use N (integer), or
FixedPoint.newFromRaw(raw_fp12) for fractions. At export we auto-rewrite
decimal literals; if you're seeing this, the rewriter didn't match
this token.)`.

Low lift, huge quality-of-life win for the day the rewriter misses
a corner case.

#### Task 3 — PS1 Lua authoring cheatsheet (one page)

`docs/lua-ps1-cheatsheet.md` — a dense one-pager with:
- Y is inverted (+Y = down, -Y = up). Diagram.
- `Player.GetPosition()` in third-person returns the camera head,
  *not* the player body. The rig-back distance is 3 world units.
  Body ≈ `playerPos + fwd * 3`; chest ≈ `playerPos.y + 1`.
- fp12 crash course — integer vs FixedPoint vs raw, when APIs want
  each.
- Edge-triggered `Input.IsPressed` vs level-triggered `Input.IsHeld`.
- Sample scripts that apply each convention correctly (pull from
  `combat_showcase.lua`).

Pure docs, no code. Cheapest item; ship alongside the above as
authoritative reference.

#### Task 4 — EmmyLua stub generator

ROADMAP #4 done properly. Parse `luaapi.hh` doc comments (they're
already structured — `// Entity.Spawn(tag, {x,y,z} [, rotY]) ->
object or nil`), emit a `.lua` stub with `---@param` / `---@return`
annotations Rider/VS Code picks up.

Scope bigger than Tasks 1-3; likely an afternoon of work. Defer
unless Tasks 1-3 finish fast.
