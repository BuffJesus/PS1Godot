### 1. `Entity.SetPosition` signature was wrong (FIXED but not pushed)

`test_logger.lua` was calling `Entity.SetPosition(playerMesh, p.x, p.y, p.z)`
— the runtime takes a Vec3 table as the second arg, not 3 numbers.
Silent no-op via `if (!lua.isTable(2)) return 0;` left the player
mesh stuck at (0, 0, 0) regardless of runtime player position.

Fixed in `test_logger.lua` to `Entity.SetPosition(playerMesh, p)`. Stage
+ commit + push first thing.

### 2. Player mesh "way too large" on PSX

Humanoid renders at sane size in Godot's editor viewport but is huge
on PSX. Hypothesis: the PSXMesh exporter reads raw vertex positions
from `mesh.GetSurfaceArrays()[Vertex]` and divides by `gteScaling=4`,
but **doesn't apply the MeshInstance3D's transform scale**. If the FBX
imported with a non-1.0 internal scale (e.g. cm units → factor 100)
that's compensated by an instance transform scale on the Godot side,
the exporter sees the raw (huge) vertex values.

Fix candidate: in `PSXMesh.FromGodotMesh()`, multiply each vertex by
`node.Scale` (or `node.GlobalTransform.Basis.Scale`) before the
gteScaling divide. Plus emit a warning when `Scale != Vector3.One`
since combining that with affine warp can look weird.

File: `godot-ps1/addons/ps1godot/exporter/PSXMesh.cs:17` —
`FromGodotMesh()`.

If audio still silent after (1) is fixed: check whether per-clip

### 4. Camera "wasn't attached to player"

## Important context to remember

- **psxlua is integer-only.** No float literals (`0.00001`, `2.4`).
  Use `Convert.IntToFp(rawValue)` if you need fractional fp12 values.
- **tscn does NOT support `#` comments.** Adding `#` lines corrupts
  the parser and can drop nodes silently. Notes go in commit messages
  or adjacent .md files.
- **Entity.SetPosition takes a Vec3 table.** Same for any other
  function that takes a Vec3 — don't pass 3 numbers.
- **Default material `ps1_default.tres`** must stay texture-free. It
  gets auto-assigned to every new PS1MeshInstance via
  `_EnterTree()`. Floor uses the dedicated `ps1_floor_grass.tres`.
- **Cutscene.Play vs PS1AudioEvent timing.** The runtime's
  `m_frame` and Lua's onUpdate-tick advance at different rates
  (60 fps onUpdate vs 30 fps cutscene-frame). For text+audio sync
  during cutscenes, fire BOTH from Lua at the same `cutsceneFrame`
  counter (which only increments while `Cutscene.IsPlaying()`).
  Don't use PS1AudioEvent + separate Lua text reveals.
- **Demo layout right now**:
  - PS1Player spawn: `(0, 2, 8)` facing -Z
  - PS1TriggerBox: `(0, 1, 4)` HE=(3, 2, 1) — gate
  - Cube (green, spinning): `(-4, 1, -2)` interactable Triangle
  - Cube2 (checkered, bouncing): `(4, 1, -3)` interactable Triangle
  - CheckeredZone trigger: wraps Cube2
  - Player (humanoid, T-pose): tracks runtime player position via Lua

## Follow-up session, evening 2 — sequenced music + cutscenes + rooms + multi-scene

A second long session immediately after the items above. Everything
below shipped on `main`. Headlines:

**Phase 2 bullet 12 (rooms/portals) — landed end-to-end on PSX.**
- `PS1Room` + `PS1PortalLink` authoring nodes; exporter assigns
  triangles to rooms by vertex-majority containment with a 0.5 m
  boundary expand, resolves portal rooms via NodePath, auto-corrects
  portal normal to point RoomA → RoomB. Catch-all room appended for
  unassigned triangles.
- Diagnosed a misleading runtime safety fallback in `renderer.cpp`'s
  `RenderWithRooms`: when `cameraRoom < 0`, the loop renders ALL rooms
  unconditionally, hiding portal culling completely whenever the
  player is in the catch-all (very common in partial-interior scenes).
  Local patch removes the fallback; tracker entry filed in
  `docs/psxsplash-improvements.md` (entry N+6).
- Demo wires two rooms (`Room_A` / `Room_B`) connected by a
  `PortalLink_AB` to demonstrate the path; from the catch-all
  exterior, both rooms cull correctly.

**Phase 2 bullet 10 polish — cutscene rotation convention solved.**
- Spent multiple iterations chasing a "camera points wrong direction"
  bug in the intro cutscene. Root cause was on the exporter side:
  `SceneCollector.EncodeKeyframeValue` used `DegToFp10 = 4096f / 360f`
  for rotation tracks, but `psyqo::Angle` is `FixedPoint<10>` measured
  in fractions of Pi (1.0 pi-unit = 180° = 1024 raw fp10). Every
  authored angle was being doubled. Fix: `1024f / 180f`.
- Documented the matrix convention while debugging (entry N+1 in
  `psxsplash-improvements.md`): `psyqo::Matrix33::vs[i]` is column i,
  `Camera::SetRotation` builds `M = rotY * rotX * rotZ`, and
  `worldToCamera` uses `vs[2]` as the camera-forward direction.
  Saved future cutscene authors from re-deriving this from psyqo's
  source.
- Cutscene now arcs the camera + zooms on the green cube during
  narration, hands back to the player rig cleanly.

**Multi-scene support landed.**
- `PS1Scene.SubScenes` (`Array<PackedScene>`) — additional scenes
  packed alongside the main scene as `scene_1.splashpack`,
  `scene_2.splashpack`, etc.
- `Scene.Load(N)` Lua API requests a scene transition; processed at
  the end of the current frame in `processPendingSceneLoad`.
- `Scene.GetIndex()` returns the currently loaded scene.
- Demo wires a "checkered realm" sub-scene with a teleport trigger
  (`teleport_to_realm.lua`) and a return trigger (`teleport_back.lua`).
- Discovered + worked around a Godot orphan-tree bug: sub-scene
  exports gave (0,0,0) positions for everything because Godot gates
  `GlobalTransform` on `is_inside_tree()`. Fix was to park the
  instance under `SceneTree.Root` during export.

**Dialog system rewrite — single-line per press, audio-aware hide.**
- Multi-line auto-advancing dialog had wedge cases that left the box
  stuck on screen. Replaced with a simpler single-state model: each
  Triangle press shows one line, schedules a hide tick, repeats.
- Added `Audio.GetClipDuration(nameOrIndex)` Lua API — derived from
  ADPCM size + sample rate. Dialog scripts now use
  `hideAtTick = tick + max(MIN_HOLD, audioFrames + TAIL)` instead of
  a flat 1.5 s timer, so the box stays up for the full voice clip
  and not a split second longer.
- `Audio.GetClipDuration` exposed via `AudioManager::getClipDurationFrames`.
- All three dialog scripts updated (`test_logger.lua`,
  `checkered_dialog.lua`, `realm_cube_dialog.lua`).

**Sequenced music — ground-up implementation.**
- New custom binary format (PS1M) parsed at export from MIDI, played
  back by a new `MusicSequencer` class on top of psxsplash's
  `AudioManager`. Format documented in
  `docs/sequenced-music-format.md`; runtime patch tracker entry filed
  as N+7 in `psxsplash-improvements.md`.
- Splashpack format bumped **v21 → v22** (header tail +8 bytes for
  music table offset/count). Loader assert bumped to `>= 22`.
- Authoring side: `PS1MusicSequence` resource (picks `.mid`, holds
  `Channels` array of `PS1MusicChannel` sub-resources), exposed via
  `PS1Scene.MusicSequences`. C# MIDI parser (SMF format 0/1) +
  PS1M serializer in the exporter.
- Routing: each `PS1MusicChannel` carries `MidiChannel` filter,
  optional `MidiTrackIndex` filter (for DAW exports that put every
  track on channel 0), and optional `MidiNoteMin` / `MidiNoteMax`
  range filter (for drum kits where one MIDI channel drives multiple
  sample channels — kick @ note 36, snare @ 40, hat @ 42).
- 84-entry pitch lookup, exact 12th root of 2 in fp12, capped at the
  SPU's 0x3FFF. Precomputed as static rodata; we tried iterative
  computation first and the rounding error compounded into wildly
  wrong pitches.
- Voice reservation: `AudioManager::reserveVoices(n)` +
  `playOnVoice(...)`. Sequencer claims voices `0..channelCount-1`
  for the song's lifetime; dialog/SFX use voices `[n, MAX_VOICES)`.
  No more "music note silenced because dialog stole its voice."
- Dialog auto-ducks the music: `Music.SetVolume(18)` on show,
  restored on hide. Coordinated via a `bgmMasterVol` Lua global so
  scripts share the same restore target.
- Lua API: `Music.Play(nameOrIndex, vol)`, `Music.Stop()`,
  `Music.IsPlaying()`, `Music.SetVolume(v)`, `Music.GetBeat()`,
  `Music.Find(name)`.
- Demo wires a 6-channel sequence (bass, pad, lead, kick, snare, hat)
  driven by a Reaper-authored test MIDI.

**Bug postmortems worth keeping**
- **MIDI sort order:** parser must sort NoteOff before NoteOn at the
  same tick. Otherwise `Off(60) + On(60)` at one tick silences the
  freshly-started note via the mono-per-channel cut. Half the drums
  (every snare hit) were dropping until this was fixed.
- **Tempo conversion:** runtime's `m_dt12 = 4096` represents one
  **30** fps frame, not 60. Sequencer's `ticksPerFrame12` had to
  divide `ticksPerSec` by 30, not 60. Music played at half speed
  until corrected.
- **Pitch table accuracy:** described above; bake the table.
- **Stale build copies:** `godot-ps1/build/psxsplash.ps-exe` is the
  binary the launcher loads, not `psxsplash-main/psxsplash.ps-exe`.
  Run `scripts/build-psxsplash.cmd` to copy fresh, or do it manually
  after each runtime patch. Older binary against newer splashpack
  format → silent freeze due to header-size offset drift.
- **Godot 4.4+ defaults WAV import to QOA** (lossy compressed). The
  exporter needs raw PCM; project now sets `[importer_defaults] wav =
  { "compress/mode": 0 }` so future drops don't hit this. Existing
  `.wav.import` files have to be patched + cached `.sample` blobs
  cleared.
